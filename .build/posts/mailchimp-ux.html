User Experience

Overview

Mailchimp is a popular and very well-designed email marketing service. Considering the large number of Mailchimp users, this project aims at maximizing user experience by making minor tweaks, so users can easily adapt to the changes. The case study documents the full process of this project which ends in May 2014.

Roadmap

Business goals

  • Have more users choose MailChimp over competitors;
  • Convert free users to long-term, paid users

Project goals

  • Optimize user flow, reduce back and forth;
  • Make features and strengths more obvious;
  • Let users get the most out of the service;
  • Make MailChimp functions easier to understand.

Major competitor

Target audience

Potential customers who are testing multiple email marketing services to find the best fit.

Methods to identify problems

  • Gather user feedback

List of problem assumptions

  • Cannot visualize what product looks like instantly
  • Do not know meaning of terms
  • Too much text
  • Cannot see certain examples
  • Not easy to find tips and tricks to maximize ROI
  • Navigation is not clear
  • It is not clear to potential customers why Mailchimp is better than competitors
  • Cannot name campaign from the start
  • Difficult to create or customize mailing lists

Create user tests

Purpose

I've created 2 user tests: a click test and a comparative test, to validate assumptions and discover new problems.

Sample tasks and questions

  • First impressions
  • Create a test campaign
  • Edit image
  • Navigate around the website

Conduct user tests

  • Method: shadowing and interview afterwards
  • 4 participants
  • 2 tests, including 1 comparative test
  • Average length of test: 21 minutes
